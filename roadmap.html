How far MyBI has come

MyBI ships usable software at every step. The core platform, data tooling, Calc Engine, a 40-kind chart catalogue, the dashboard canvas, analytics, publishing and the on-device AI assistant are already in your hands — here's the full picture, phase by phase.

Milestone 1 — Foundations & data

A working app shell, project system and a complete pipeline for getting data in and shaping it.

Phase 1

Core platform

✓ Shipped

App shell, workspace/project system, the Importer for local files, the data explorer, and the embedded SQL engine + local metadata store.

Phase 1.1

.mybi bundle + app shell

✓ Shipped

Folder-bundle project format, an activity bar of blocks, the folder explorer panel, drag-file-to-canvas, and the typed block event bus.

Phase 1.2

Importer connectors

✓ Shipped

SQL databases (read-only) and REST / JSON connectors, with a connection manager and passwords stored in the OS keychain.

Phase 1.3

Get-Data wizard + Data editor

✓ Shipped

A guided Get-Data wizard with live preview, merge-across-files & sheets, a spreadsheet-style transform ribbon (group by, pivot, window, rank…), revertable history, quality tools, and export to CSV / TSV / JSON / Parquet / XLSX. Your original file is never modified.

Phase 1.4

Workspace shell — panels, rail & settings

✓ Shipped

An activity-bar shell with a mirrored right context dock (Data / Charts / Analytics / AI), capped on-screen panels, pinning, theming (System / Light / Dark / OLED) and macOS trackpad haptics. The Settings area was rebuilt into App / Project / Dashboard scopes, with Project subsystems — git remote, encrypted-forensic export, plugin permissions and AI provider keys.

Phase 1.5

Data warehouse — multi-source modelling

Planned

Org-scale ingestion, incremental append, schema-drift handling and scheduled refresh — moving to a separate companion app MyBI connects to via a Warehouse connector.

Milestone 2 — Modelling & the compute core

One engine that owns every number, with relationships and a formula language on top.

Phase 2

Semantic block — relationships

✓ Shipped

A visual model diagram: draggable table boxes, drag-to-connect, and data-verified auto-detect with cardinality + confidence, powering cross-table joins.

Phase 2.1

Calc Engine — single compute core

✓ Shipped

One shared core: a QuerySpec compiled to a single canonical SQL query (aggregations, % of total, stats), parameterised filters and golden-number consistency tests.

Phase 2.2

MyBI formula language

● In progress

A safe expression compiler with a Data-editor formula editor, live validation, autocomplete & ~80 engine-verified row-wise functions and table joins. Next: aggregate functions, dependency graph and reusable named measures.

Milestone 3 — Visualise & analyse

Turn results into charts on a dashboard canvas, with an analyst toolset alongside.

Phase 3

Charts block

✓ Shipped

Two native chart engines with 40 chart kinds — from bar, line and pie to heatmap, sankey, waterfall, calendar, geo maps and 3D — with a live-preview builder, a per-chart options popup and animated per-kind explainers. Every chart computes through the Calc Engine.

Phase 4

Canvas block

✓ Shipped

A zoomable dashboard canvas (fixed sizes or freeform) with rulers, a full-featured ribbon, free drag-resize cards, cross-filtering, multi-sheet footer tabs, page-builder chrome (header / footer / nav bars), slicers, metric tiles, shapes & SmartArt, palette-driven themes and per-dashboard layout persistence.

Phase 4.1

Analytics block

✓ Shipped

An Analytics tab on charts: trend lines, forecasting, confidence intervals and correlation, with configurable parameters — all computed through the Calc Engine.

Phase 4.2

Sentiment analysis

Planned

A text / sentiment toolset wired through the Calc Engine and the Analytics tab.

Phase 4.5

Import-adapter framework

● In progress

An ImportAdapter contract + canonical ImportedReport model so dashboards exported from other BI tools can later plug in and re-render on the MyBI canvas. Contract stubbed; parsers come in Phase 13.

Milestone 4 — Share & version

Publish dashboards and treat every project as versioned, git-native history.

Phase 5 · 5.1

Publish & share

✓ Shipped

Three ways to share a dashboard: live from your own self-hosted server (scheduled refresh + live views over your sources), a portable read-only encrypted .mbix snapshot, or a static aggregated web page. In all three, only aggregated results ever leave — raw rows never do. Multi-page printable reports (PDF / PNG / spreadsheet) are still to come.

Phase 5.2

Git versioning + push to a remote

✓ Shipped

The project folder is a git repo — each dataset commit is a git commit. A Data → Git terminal, since-commit Explorer badges, U/M tracking badges and push to a git remote, with Project → Git settings, structured commits and in-app sign-in to your git host.

Milestone 5 — Extensibility & the locked platform

Expose the block API to third parties through a versioned SDK — with untrusted code sandboxed behind declared permissions.

Phase 6

Plugin / extension framework

● In progress

Underway: plugin loading with tiered rails (first-party · Trusted by MyBI · community), Ed25519 signing & verification, a host field drag-and-drop bridge, plugin-contributed panels & visuals, and per-plugin permission manifests surfaced in Settings. Still to come: full sandbox & resource enforcement (6.3 / 6.4).

Phase 6.1 · 6.2

Web scraper & marketing connectors

Planned

Basic scraping to CSV / JSON / XLSX, plus Google Analytics, Facebook and X (Twitter) analytics data sources.

Phase 6.3 · 6.4

Permission, sandbox & audit

Planned

Runtime permission checks on every call, capability gating with RLS/CLS + masking before data leaves the core, resource controls, network-off-by-default, and an immutable audit log across the extension lifecycle.

Phase 6.5

Palette marketplace

Planned

A colour-palette marketplace mirroring the plugin flow — Installed / Marketplace / My palettes, signed like plugins, theming the app and every chart.

Milestone 6 — Geospatial, modelling & scripting

Professional-grade GIS, simulation, and governed Python / R / query scripting.

Phase 7 · 8

GIS foundation & advanced GIS

● In progress

Point, heat & choropleth maps (with 3D views) already ship as chart kinds. Next: full GIS data formats — GeoJSON / Shapefile / GeoPackage / KML / GPX, spatial databases / WMS / WMTS / XYZ — then spatial analysis, raster, cartography and print layouts.

Phase 8.5

Simulation & modelling

Planned

User-defined criteria driving business/financial, statistical/ML, operations and system-dynamics models — all computing via the Calc Engine.

Phase 8.6

Scripting block — Python / R / query

Planned

Scripts as analytical consumers of governed datasets (platform.get_dataset(…)), run in sandboxed runtimes — never connecting to sources or credentials themselves. Built on the Phase 6 SDK + permission layer.

Milestone 7 — AI, marketplace & the cloud

Conversational analysis, hosted publishing, a marketplace, format compatibility, and finally accounts.

Phase 9

AI assistant

✓ Shipped

A native, on-device assistant: chat about your datasets and documents (built-in retrieval), pluggable local engines, guardrails with author-tagged actions you approve, and consent-gated data access — nothing is sent to a cloud, and raw values only flow with your explicit OK.

Phase 10 · 11

AI GIS & the API builder

Planned

Natural-language spatial SQL and map generation, plus a REST API builder over published dashboards.

Phase 12

Marketplace

● In progress

Installing signed plugins & palettes from the in-app marketplace is live. Next: publishing, discovery, reviews, versioning and dependencies for blocks, extensions and script packs — surfacing each item's declared permissions before install.

Phase 13

Compatibility layer

Planned

Format adapters that open dashboards and projects exported from other BI and GIS tools and re-render the equivalent dashboard on the MyBI canvas.

Phase 14

Accounts, organisations & roles

Planned

Opt-in: a hosted workspace with organisations, roles (developer / editor / viewer), SSO / OAuth / SAML, audit logs, lineage and RLS/CLS.

Phase 15

Notarised distribution & own-domain email

Planned

Apple-notarised, OS-signed installers with listings on the Mac App Store and Microsoft Store, plus a move from the interim address to MyBI's own-domain email and support.

Be there for the next phase.
